# RoPE 系列原理解析

## 来源信息
本来源摘要整合了多篇关于 Transformer 位置编码，特别是旋转位置编码（RoPE）的核心推导与演进思考文章：
1. **Transformer位置编码基础详解**：探讨绝对与相对位置编码的各种实现。
2. **Roformer：旋转位置编码的核心原理与实现**：RoPE 的首次提出，推导绝对位置如何转化为相对位置效果。
3. **RoPE局限性与多模态扩展思考**：探讨长序列的衰减问题以及扩展到 2D 的初始构想（RoPE-Tie）。
4. **“闭门造车”之多模态思路浅谈（三）：位置编码**：确立多模态位置编码的三个核心原则（兼容性、等价性、对称性）。

## 核心要点摘要

### 1. Transformer 位置编码的发展
- **绝对位置编码**：如训练式（BERT）、三角函数式（Sinusoidal）。形式简单，但往往缺乏良好的长度外推性。
- **相对位置编码**：如经典式（Shaw et al.）、T5式（仅加偏置项）、DeBERTa式。通过在 Attention 矩阵中引入相对距离的函数来工作，通常效果更好。
- **RoPE 的提出**：苏剑林提出将位置信息映射到复平面，利用复数乘法的旋转特性，将绝对位置注入到 $Q$ 和 $K$ 中，而在内积计算时自动体现为相对位置。它不仅外推性好，且能兼容线性 Attention。

### 2. 二维与多模态扩展的推导
- **2D-RoPE 的理论基础**：在视觉任务中，图像是二维的 $(x, y)$。通过矩阵指数运算推导，二维 RoPE 的通解是分块旋转矩阵，即把输入向量一分为二，一半施加 $x$ 的一维 RoPE，一半施加 $y$ 的一维 RoPE。
- **多模态统一挑战**：
  - **向后兼容**：多模态位置编码应能退化为对应的一维/二维 RoPE。
  - **等价与对称**：文本 Token 与图像 Patch 之间应有合理的偏置（如 RoPE-Tie），保证夹在文本中间的图像在相对位置上的连贯性。
  - **三维的困境**：视频引入了时间维度（T）。时间维度由于具有“不断推进”的特性，不应与空间维度（H, W）完全平权处理。
