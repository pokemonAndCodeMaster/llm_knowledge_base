# 视觉基石与卷积原理

## 来源信息
本来源摘要整合了多篇关于计算机视觉基础模型、卷积操作以及对比学习多模态编码器的长文，构成了多模态大模型的视觉入口知识：
1. **ViT(2020, Google Research)**：介绍了原始 Vision Transformer 的架构，将图像切割为 Patch 并输入 Transformer，打破了 CNN 的垄断。
2. **ViT参数量、显存计算**：详细推导了 ViT 在各层的参数量（以 ViT-Base 86M 为例）以及训练/推理阶段的显存（Activation）开销公式。
3. **ViT的位置编码**：原始 ViT 中的可学习绝对位置编码原理（相加作为偏置）。
4. **CLIP(2021, OpenAI)**：开创性的对比学习图文对齐模型，阐述了通过大规模图文对构建统一高维语义空间的原理与代码流转。
5. **SigLIP / SigLIP2**：Google 提出的替代 CLIP 的高效损失函数方案。将全图 Softmax 优化为每个 Pair 独立的 Sigmoid 损失，去除了全局归一化，支持 chunked 极致切分训练，并融合了自蒸馏和掩码预测。
6. **深度学习中不同类型卷积的综合介绍**：全面梳理了 2D卷积、3D卷积、转置卷积、空洞卷积、可分离卷积等的区别。
7. **【Torch】nn.Conv1d、nn.Conv2d、nn.Conv3d算法详解**：从 Torch 算子层面严谨给出了不同维度卷积输出形状的数学计算公式（包含 stride, padding, dilation 的影响）。

## 核心要点摘要

### 1. ViT 的破局与计算
原始 ViT 核心是将图像转换为序列。
- **Patch Embedding**: 使用卷积将图像物理切块并映射到 `embed_dim`。
- **CLS Token**: 追加的可学习向量，用于最终的全局分类。
- **绝对位置编码**: 将一个与序列等长的 1D 可学习张量直接加在输入上。
- **参数量分配**: ViT 的参数大头在 12 层的 Attention 和 MLP (通常隐藏层扩张 4 倍) 中。训练时，混合精度（fp16）虽加速计算，但在内存里依然需要存留 float32 副本，实际显存开销并未同比例减半（这纠正了常见误区）。

### 2. 视觉特征提取器：CLIP 与 SigLIP 家族
- **CLIP**: 使用对比学习（Contrastive Learning），在一个 Batch 内将 N 张图和 N 段文字提纯，算 $N \times N$ 的相似度矩阵，对角线为正样本，通过交叉熵拉近图文距离。赋予了模型 Zero-shot 迁移能力。
- **SigLIP**: 将原本需要 $O(B^2)$ 内存的全局 Softmax 优化成 Sigmoid，每个配对独立做二分类，使得可以按 Block 传输数据计算损失，极大地突破了 Batch Size 限制。
- **SigLIP2**: 在原版基础上引入了 LocCa（解码器预训练定位）、SILC 和 TIPS（掩码预测和自蒸馏），赋予了模型稠密（Dense）局部特征认知能力，使其能充当各类细粒度下游视觉任务的万能编码器。

### 3. 卷积算子机制
无论是 `Conv1D`, `Conv2D` 还是 `Conv3D`，其本质都是用一个小的滑动窗口与输入进行互相关（点积）计算。
在多模态预处理和早期的切块器中，`Conv3D` 被巧妙地用作**时空联合压缩器**（同时在视频帧的时间轴 T、高度 H、宽度 W 上滑动）。计算公式严格遵循 `[(L + 2*P - D*(K-1) - 1) / S] + 1` 的下取整规律。
